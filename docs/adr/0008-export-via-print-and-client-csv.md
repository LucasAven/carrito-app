# Export is the current Balance scope, as browser-print PDF and client-built CSV

Export exists for one job: handing the books to an accountant (see [v2-wishlist](../v2-wishlist.md)). It scopes to **what the Operator is already looking at**, not a separate date-range picker: the "Exportar" menu item carries the active Balance query params (`date` | `week` | `month` | `year` plus `payment_type`) to a new `/exportar` route, and that route resolves them through the very same [`loadBalanceEntries`](../../lib/balance/load.ts) and `getPeriodLabel` the ledger uses (both extracted from `app/balance/page.tsx` so the report and the ledger can never disagree about what a period contains). So "export" always means "export this view", which is also why opening `/exportar` with no scope falls back to today, mirroring the ledger's own default. **PDF uses no library**: the report is server-rendered HTML and "Guardar PDF" just calls `window.print()`, with a `@media print` block in `globals.css` that strips the fixed app chrome (Appbar, BottomNav, toasts) and the on-screen toolbar, undoes the shell's nested `overflow-y-auto` scrollers (which would otherwise clip a long report to the first page), and repeats the table header per page. This keeps the [fluffless, no-new-deps](../../readme.md) posture and rides the platform's own "Save as PDF" / share-to-print on both iOS and desktop. **CSV is built client-side** by [`entriesToCsv`](../../lib/export/csv.ts) (RFC-4180 escaping, Spanish headers in the [domain terms](../../CONTEXT.md), amount as a raw two-decimal number so a spreadsheet reads it as a number) and delivered through the Web Share API when the platform can share files (`navigator.canShare({ files })`), so on a phone it goes straight to WhatsApp/email; otherwise it falls back to an anchor download. A leading UTF-8 BOM is prepended to the Blob so Excel renders the accents (`Café`, `José`). The report renders in fixed light colors rather than theme tokens, so a dark-mode Operator still prints dark-ink-on-white and the on-screen preview reads as the document it becomes. We rejected a PDF library (`jspdf`/`pdfkit`): it is real bytes and layout code for a sharing convenience the browser already does well. We rejected a separate export date-range UI: it would be a second, drift-prone source of "which entries", when the ledger already answers that. We rejected a server-generated file (a Route Handler streaming CSV/PDF): the data is already in the page, the client path needs no new endpoint or auth surface, and Web Share beats a server download on mobile. Revisit the library only if an Operator needs a fixed, pixel-stable PDF layout that the print dialog's margins and page breaks can't deliver.
